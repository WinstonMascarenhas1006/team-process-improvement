## Requirement mapping

| Typical JD theme | My evidence | Verifiable metric / artifact |
|------------------|-------------|------------------------------|
| Facilitate ceremonies & team rhythm | Daily standup, capped planning, retro with one action | Planning 150 min → 90 min · [case.json](../data/cases/student-peer-review/case.json) |
| Remove impediments | Shared log + owners; CI blocker closed in daily | Open impediments at sprint end: 2 → 0 |
| Delivery predictability | Sprint goals, points, summaries before demo | Sprint completion 62% → 100% |
| Quality / Definition of Done | Team DoD checklist after retro | [working_agreement.md](../templates/working_agreement.md) |
| Stakeholder alignment | AC on stories before planning | Stakeholder quotes in dashboard |
| Continuous improvement | Compare to ING / Spotify / GDS public agile stories | Compare tab in dashboard |

- **Numbers:** Sprint 1 → 3 in `case.json`; KPI cards match the sprint table.
